### What we are porting

**Source**: `https://raw.githubusercontent.com/ionutdecebal/rsvpnano/main/src/reader/ReadingLoop.cpp` (algorithm) and `https://raw.githubusercontent.com/ionutdecebal/rsvpnano/main/test/test_pacing/test_main.cpp` (reference assertions — these become our Dart test contract).

**License**: MIT. Algorithm constants and tier thresholds may be reimplemented in Dart with attribution. Add a file-level dartdoc citation in `word_duration.dart` and a row in `NOTICE` or `THIRD_PARTY.md`.

**Algorithm summary** (verify against the C++ source — these numbers are the contract):

- **Base interval**: `60000 / wpm` ms.
- **Length bonus** (capped at 170%):
  - readable >6 chars: +6%/extra char (tier 1)
  - readable >10 chars: additional +9%/extra char (tier 2)
  - readable >14 chars: additional +12%/extra char (tier 3)
  - compound joiner (`-` between word chars): +14%; if readable ≥10 with joiner, +long-compound bonus
  - tech connectors (`_`, `/`, `.` in tokens like `U.S.` or `foo_bar`) beyond joiner count: +8% each
- **Complexity bonus**:
  - Approximate syllable groups via vowel-group counting; silent-`e` decrement when ending in `e` and not `le`/`ye`. Groups >2 → +10% per extra group, capped.
  - All-caps token (uppercase==letterCount, letterCount>0): +14%
  - Mixed letter+digit token: +15%
  - Dense connectors: additional bonus
- **Punctuation pause** (trailing rhythm char, after stripping closing quotes/parens):
  - `,` → +45%
  - `;` or `:` → +80%
  - `-` → +60%
  - `.` → +135% (suppressed if abbreviation detected)
  - `!` or `?` → +150%
  - `...` (ellipsis) → +110%
- **Abbreviation suppression** for `.`:
  - Known list: `Mr.`, `Mrs.`, `Ms.`, `Dr.`, `St.`, `Jr.`, `Sr.`, `vs.`, `etc.`, `i.e.`, `e.g.`, `U.S.`, `U.K.`, `a.m.`, `p.m.` (verify full list against C++ source)
  - Dotted initialism: token has interleaved letters and dots (e.g. `U.S.`)
  - Short-word-period: `readableLength <= 4` AND `nextWord` starts with a lowercase letter (including extended-Latin lowercase)
- **Per-category scaling** (user-controlled, 25–200%, default 100%):
  - `bonusMs = (bonusPercent * scalePercent / 100) * delayMs / 100`
  - Three independent scales: `longWordScalePercent`, `complexWordScalePercent`, `punctuationScalePercent`
  - Three independent delay knobs (advanced, default 200ms each): `longWordDelayMs`, `complexWordDelayMs`, `punctuationDelayMs`
  - Scale floor is 25% (a 0% setting clamps to 25%) — preserves minimal pacing variation

The source-of-truth for exact numbers is the **ported Dart test suite**, which mirrors `test_main.cpp` line-for-line. If your implementation makes a test fail, the implementation is wrong, not the test.
