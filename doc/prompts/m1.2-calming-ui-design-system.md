**Feature F1.2.1.1 — Color Token System**

#### Task T1.2.1.1.1.1 — Create RunThruColors with 4 theme palettes
- **Files in scope**: `lib/shared/design/colors.dart`
- **Files out of scope**: `lib/design/tokens.dart` (keep existing — coexist)
- **What to do**: Create a new `RunThruColors` class defining semantic color tokens for 4 themes: Light Warm, Dark, Sepia, High Contrast. Tokens should include: `surface`, `onSurface`, `surfaceVariant`, `primary`, `secondary`, `accent`, `error`, `stageSurface`, `stageText`, `stageAnchor`, `shadow`, `shadowLight`, and any reading-specific colors. Use CVD-safe blue+orange pair with lightness differentiation.
- **Constraints**:
  - All text/background pairs must meet WCAG 2.1 AA (≥4.5:1 normal text, ≥3:1 large text)

## Hard Rules (from CLAUDE.md — never violate)

1. All colors via design token classes — no raw `Color(0xFF...)` in widgets
2. All text styles via typography tokens — no hardcoded `TextStyle`
3. All box shadows via decoration factories — no hardcoded `BoxDecoration` shadows
4. Every animation checks `isReducedMotion(context)`
5. Riverpod for state — no `setState()` for shared state
6. go_router only — no `Navigator.push()`
7. No deprecated Flutter APIs. `ListenableBuilder` over `AnimatedBuilder`
8. pdfrx on main isolate only (FFI)
9. No color-only signals — shape/label/position fallback on every indicator
10. Touch targets: ≥44pt iOS, ≥48dp Android
11. CVD-safe defaults (blue+orange + lightness). Tested against protanopia/deuteranopia/tritanopia
12. Accessibility features NEVER paywalled (adaptive spacing, OpenDyslexic, ruler, CVD themes, font size)
13. All public symbols have dartdoc comments
14. Conventional commits: `feat/fix/chore/docs/test/refactor`

## Important Notes

- **Coexistence strategy**: New `lib/shared/design/` files coexist with old `lib/design/` files. Screens being redesigned in E1.2.2 switch to the new tokens. Screens NOT touched in this milestone keep using old tokens. Full migration happens incrementally in later milestones.
- **Generated files**: After creating `@riverpod` providers, run `dart run build_runner build --delete-conflicting-outputs` to generate `.g.dart` files.
- **Font licensing**: Atkinson Hyperlegible is SIL OFL. OpenDyslexic is free/open-source. Include LICENSE files.
- **Do NOT touch** the ORP engine (`lib/core/orp.dart`), 3D viewport (`lib/three_d/`), or gesture system. Those are out of scope for this milestone.
- **reading_screen.dart and parallax_reading_screen.dart** use `ConsumerStatefulWidget` with animation controllers — they may need `HookConsumerWidget` (with `useAnimationController`) rather than plain `ConsumerWidget`. Use hooks where animation lifecycle management is needed.
- **settings_screen.dart** is already `ConsumerWidget` — the migration task is about its child `StatefulWidget`s (`_ParallaxIntensitySelector`, `_ReadingGoalSelector`) which use `setState()`.
