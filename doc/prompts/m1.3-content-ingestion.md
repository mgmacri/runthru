### Key Data Model

All extractors produce `ExtractedDocument` (from `lib/services/models.dart`):
```dart
class ExtractedDocument {
  final List<Sentence> sentences;
  final List<PageBoundary> pageBoundaries;
  final int totalPages;
}

class Sentence {
  final List<String> words;
}
```

This is the interface consumed by the reading screen / word timer. Any new content source must produce an `ExtractedDocument`.

## Hard Rules (from CLAUDE.md — never violate)

1. **All colors** via design token classes — no raw `Color(0xFF...)` in widgets
2. **All text styles** via typography tokens — no hardcoded `TextStyle`
3. **All box shadows** via decoration factories — no hardcoded `BoxDecoration` shadows
4. **Every animation** checks `isReducedMotion(context)` and applies reduced-motion override
5. **Riverpod for state** — no `setState()` for shared state
6. **go_router only** — no `Navigator.push()`
7. **No raw Material loading widgets** — no `CircularProgressIndicator`, `LinearProgressIndicator`, `RefreshIndicator`
8. **Heavy computation in Isolates** — never on the main event loop
9. **No deprecated Flutter APIs** — `ListenableBuilder` over `AnimatedBuilder`
10. **pdfrx on main isolate only** (FFI constraint). Classification in `Isolate.run()`
11. **No color-only signals** — shape/label/position fallback on every indicator
12. **Touch targets**: ≥44pt (iOS) / ≥48dp (Android)
13. **CVD-safe defaults** — blue+orange with lightness variation
14. **Accessibility features NEVER paywalled**
15. **All public symbols** have dartdoc comments
16. **Conventional commits**: `feat/fix/chore/docs/test/refactor`
17. **Privacy by default** — reading content on-device only. No cloud upload without per-session opt-in
18. **No dark patterns** — clipboard auto-detect must NOT auto-import. Explicit user confirmation required.

## Privacy Guardrails (Critical for M1.3)

Content ingestion is the highest-privacy-risk milestone:

- **Clipboard**: Read on foreground for DETECTION only. Import requires explicit tap. Never log clipboard contents.
- **Share intent**: Content copied to app-private storage immediately. Source URI permissions released.
- **File picker**: Files copied to app-private storage. Original paths not retained.
- **No analytics on content**: Do not track what users read, import, or share. Session-only metadata at most.
- **No cloud upload**: All content stays on-device. Period.

## Important Notes

- **Generated files**: After creating `@riverpod` providers, run `dart run build_runner build --delete-conflicting-outputs` to generate `.g.dart` files. Never edit `.g.dart` files manually.
- **Kotlin file path**: `MainActivity.kt` lives at `android/app/src/main/kotlin/com/speedyboy/speedy_boy/` (old filesystem path) but its package declaration is `com.runthru.app`. Edit the file at its current path.
- **Existing `ClipboardDocument`**: Lives at `lib/core/clipboard_document.dart`, NOT `lib/services/`. The content normaliser should reuse its sentence/word tokenisation logic (DRY principle) rather than reimplementing.
- **`file_picker` is already in pubspec**: No need to add it — just import and use it.
- **iOS Share Extension**: This is a separate Xcode target with its own build settings. It runs in a sandboxed process. Communication with the main app is via shared App Group file container only.
- **Do NOT touch**: ORP engine (`lib/core/orp.dart`), 3D viewport (`lib/three_d/`), gesture system, reading screen's core logic. These are out of scope.
- **Coexistence**: New content providers in `lib/features/content/` coexist with existing services in `lib/services/`. The new providers wrap/call the existing services — they don't replace them.
