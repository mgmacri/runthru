## New Rules (35–42)

35. **Assistant model never bundled with app binary.** The GGUF model file is downloaded after install via `ModelDownloader`, stored in `<appSupport>/assistant_model/<variant>/`, and verified by SHA-256 against the remote manifest. Never check model files into source control. Never include them in the Flutter asset bundle. The app binary growth from v6 to v7 must stay under 10 MB.

36. **Assistant prompts and responses never logged.** Prompt text, sentence context, and model output must never appear in any log, crash report, analytics event, or persistent storage beyond the lifetime of the AssistantSheet session. Log only metadata: prompt length, response length, latency, model variant. Crash report payloads must scrub the full assistant context. This is non-negotiable per P19.

37. **No background inference. No prefetch. No suggestions.** The assistant runs only when explicitly summoned by the user via the documented gesture (long-press in reading area while paused). Never warm up the model on app launch. Never pre-generate responses for likely questions. Never surface unsolicited assistant content. Per P21.

38. **Inference runs only on the Rust bridge.** All llama.cpp invocation lives in `rust/src/assistant.rs`. Dart calls go through `AssistantService` → `flutter_rust_bridge` → Rust. Never call into native code from Dart directly. Never bypass `AssistantService` to invoke the bridge from a widget.

39. **Model load is once-per-session, lazy.** The first call to `AssistantService.complete()` triggers `model_loader.load()` in Rust (mmap-backed, ~200ms). Subsequent calls reuse the loaded model. Never reload the model per query. The loaded model must be released only when the app is backgrounded for >5 minutes (memory pressure) or the user disables the assistant in Settings.

40. **Assistant context is the current sentence only.** The prompt sent to the model includes exactly: a system prompt (constant, defined in `assistant_service.dart`), the current sentence from the reading viewport, and the user's question. Never send conversation history. Never send the full document. Never send the user's reading position, WPM, or any other app state. Per P20 (narrow scope).

41. **AssistantNotifier is auto-dispose.** State is reset every time the AssistantSheet closes. No persistent assistant state across reading sessions. No history view. No "previous questions" UI. The assistant is stateless from the user's perspective and should be stateless internally too. Use `riverpod-auto-dispose` patterns.

42. **Synthetic data pipeline is dev-only, lives outside `lib/`.** The dataset generation, training, and conversion scripts live in `tools/assistant/` and must never be imported by app code. The pipeline depends on Python, Anthropic SDK, Unsloth, and llama.cpp build tooling — none of these are Flutter dependencies. CI must skip `tools/assistant/` when building the app.

## Build System Changes

### Codemagic Workflow Additions
A new Rust build phase runs before the Flutter build phase on both iOS and Android workflows:

```yaml
scripts:
  - name: Install Rust toolchain
    script: |
      curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
      source $HOME/.cargo/env
      rustup target add aarch64-apple-ios aarch64-apple-ios-sim    # iOS workflow
      rustup target add aarch64-linux-android armv7-linux-androideabi  # Android workflow

  - name: Generate flutter_rust_bridge bindings
    script: |
      cargo install flutter_rust_bridge_codegen --version <pinned>
      flutter_rust_bridge_codegen generate

  - name: Build Rust library
    script: |
      cd rust
      cargo build --release --target <target-triple>
      # iOS: lipo into universal .a, copy to ios/Runner/Frameworks
      # Android: copy .so files into android/app/src/main/jniLibs/<abi>/
```

### Android NDK
- Required NDK version pinned in `android/app/build.gradle`
- `ndkVersion = "26.1.10909125"` (or current LTS)
- Add `android.bundle.enableUncompressedNativeLibs=false` to `gradle.properties` so the OS can mmap `.so` files directly

### iOS
- Set `EXCLUDED_ARCHS[sdk=iphonesimulator*] = i386` (Rust does not target 32-bit iOS sim)
- Bitcode disabled (Rust output is not bitcode-compatible by default)

## New Dev-Only Documentation

`tools/assistant/README.md` is the operator guide for the synthetic data pipeline. It is not user-facing and not part of the app, but it is part of the release process. Maintain it alongside the model. Cover at minimum:

- How to set `ANTHROPIC_API_KEY` and run `generate_dataset.py`
- Cost estimate for a full run (~$10–$25 on Sonnet 4.6 Batch)
- How to run training on the dev box (RTX 2060 6GB constraints, Llama 3.2 1B vs 3B tradeoff)
- Eval acceptance threshold (≥85% per P20 / E-14)
- GGUF conversion command and quantization choice (Q4_K_M default)
- CDN upload and manifest update procedure
- Rollback procedure if a bad model ships
